Для удаления поста необходимо сделать запрос в БД, чтобы 
получить определенный пост. Для этого нам нужен уникальный идентификатор
поста - slug

1) Создать маршрутизатор открывающий страницу подтверждения
удаления поста с динамическим slug

path('delete/<slug>', views.delete_post, name='delete_post')

2) Вид подразумевает отправку запроса для нахождения нужного поста
post_data = Post.objects.get(slug=slug)

И, после проверки метода запроса, удаления поста
(После того как юзер подтвердит удаление нажав спец.кнопку)

if request.method == 'POST':
        post_data.delete()
        return redirect('blog:home')

После удаления поста, нужно перенаправить пользователя на главную страницу
так как пост удален и оставить юзера на странице этого поста будет ошибкой


А также открытие определенной страницы   
return render(request, 'delete_post.html', {'post': post_data})

Все типы запросов в django можно сгенерировать при помощи формы
Поэтому даже при удалении нам необходима форма, которая после
нажатия кнопки подтверждения, сгенерирует и отправит запрос методом POST
